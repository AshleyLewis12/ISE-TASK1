In conclusion,online food delivery systems have made our life much easier.We can order food from anywhere by just staying at our home.For these systems to work there is a application that works behind it which manages orders,payments,restaurants and connect to delivery partners.
In this case study we get to know how the systems work and the problems that they face,might be regarding order delays,order cancellation,refund related ,payment issues and server crashes due to more load.We also compared 3 different software models to check which one is better for food delivery systems.
From our case study we got to know that Incremental Development Model is the bets choice for food delivery apps because they need  regular changes and new features.In this model we can make any changes or add new features easily.Any changes or features are added step by step here.
Overall,food delivery systems are of great use but it needs to handled and secured well.
